# 文档翻译工具使用指南

当前发布版本：`v0.4.3`

## 1. 这是什么

这是一个用来翻译单个 Markdown 文档的桌面工具。

它更适合这类场景：

- 你要翻译的是一个 `.md` 文件
- 你希望尽量保留 Markdown 结构
- 你不想覆盖原文件，而是生成一个新文件
- 你处理的是技术文档、表格较多的文档，或者夹杂路径、代码、标签的文档

支持的方向：

- `中译英`
- `英译中`

## 2. 最快上手

如果你用的是 Windows 打包版，按下面做就行：

1. 解压整个 `DocTranslationTool-win64.zip`
2. 打开 `dist\DocTranslationTool\`
3. 确认同目录下有 `DocTranslationTool.exe` 和 `.env`
4. 打开 `.env`，先只改这 3 项

```env
DOC_TRANS_BASE_URL=你的接口地址
DOC_TRANS_API_KEY=你的API Key
DOC_TRANS_MODEL=你要用的模型名
```

5. 保存 `.env`
6. 双击运行 `DocTranslationTool.exe`
7. 先看左下角“模型状态”
8. 选择一个 `.md` 文件和输出目录
9. 如有需要，手动切换翻译方向
10. 点击“开始翻译”

大多数人第一次使用，只需要这 3 个配置。

## 3. 你真正需要改什么

对大多数用户来说：

- 只需要改 `.env`
- 不需要改 `settings.json`
- 不需要改代码
- 不需要重新打包

最重要的 3 项是：

- `DOC_TRANS_BASE_URL`：你的模型接口地址
- `DOC_TRANS_API_KEY`：你的接口密钥
- `DOC_TRANS_MODEL`：你的模型名

如果这 3 项已经填好，其他项可以先保持默认。

## 4. `.env` 怎么填

### 4.1 OpenAI 兼容接口

如果你的平台是 OpenAI 兼容接口，通常这样填：

```env
DOC_TRANS_PROVIDER=openai_compatible
DOC_TRANS_API_FORMAT=openai
DOC_TRANS_BASE_URL=https://your-service.example/v1
DOC_TRANS_API_KEY=your_api_key_here
DOC_TRANS_MODEL=your_model_name
```

### 4.2 Anthropic 兼容接口

如果你的平台是 Anthropic 兼容接口，最简单的方式是这样填：

```env
DOC_TRANS_PROVIDER=openai_compatible
DOC_TRANS_API_FORMAT=anthropic
DOC_TRANS_ANTHROPIC_VERSION=2023-06-01
DOC_TRANS_BASE_URL=https://your-service.example
DOC_TRANS_API_KEY=your_api_key_here
DOC_TRANS_MODEL=your_model_name
```

这里最容易填错的是：

- `DOC_TRANS_API_FORMAT` 要改成 `anthropic`
- `DOC_TRANS_BASE_URL` 一般填服务根地址，不要自己再手动拼 `/messages`
- `DOC_TRANS_ANTHROPIC_VERSION` 一般保持默认 `2023-06-01`，除非你的服务商明确要求别的值

如果你们平台要求把 `DOC_TRANS_PROVIDER` 写成 `anthropic_compatible`，也可以；但只要当前服务是 Anthropic 兼容格式，关键还是 `DOC_TRANS_API_FORMAT=anthropic`。

## 5. 正常使用流程

1. 打开 `DocTranslationTool.exe`
2. 确认左下角模型状态不是“配置不完整”
3. 选择要翻译的 `.md` 文件
4. 选择输出目录
5. 确认翻译方向
6. 点击“开始翻译”
7. 等待批次进度跑完
8. 到输出目录查看结果文件

输出文件不会覆盖原文件。

默认命名规则：

- 中译英：`原文件名_en.md`
- 英译中：`原文件名_zh.md`

第一次使用建议先拿一个小一点的文档试一次，确认接口、模型和输出都正常，再去跑大文档。

## 6. 界面状态怎么看

你通常会看到这些状态：

- `模型状态：配置不完整`：关键配置没填全
- `模型状态：配置已加载，待检查连通性`：本地配置已经读到，但还没真正发请求
- `模型状态：正在检查接口连通性`：程序正在验证接口是否可用
- `翻译中：正在处理第 x/N 批`：当前有批次在执行
- `翻译中：已完成批次 x/N`：已经有批次返回结果，进度会继续推进

简单理解就是：

- 程序启动时只检查“配置像不像能用”
- 真正访问模型接口，是在你点了“开始翻译”之后

## 7. 大而复杂的文档怎么配

如果你翻译的是很大、很复杂、很技术化的 Markdown 文档，建议按“成功率 > 质量 > 速度”的思路来配。

建议从这组参数开始：

```env
DOC_TRANS_BATCH_SIZE=4
DOC_TRANS_PARALLEL_BATCHES=1
```

如果跑得比较稳，再慢慢加到：

```env
DOC_TRANS_PARALLEL_BATCHES=2
```

实用建议：

- 文档越复杂，越不要一开始就把并发调很高
- 表格很多、路径很多、标签很多的文档，更适合保守配置
- 如果中途失败，尽量用同一个输出目录重试，这样可以复用已缓存的已翻译片段
- 如果接口容易限流，优先降低 `DOC_TRANS_PARALLEL_BATCHES`

当前版本已经对一些常见复杂场景做了保护，例如：

- 超长表格行更安全地切分
- HTML/XML 标签保护
- 文件名、路径、技术常量保护
- 命中 `429` 时自动降并发重试
- 中断后 checkpoint / resume

## 8. 常见问题先看这里

### 8.1 显示“配置不完整”

优先检查这 3 项有没有填：

- `DOC_TRANS_BASE_URL`
- `DOC_TRANS_API_KEY`
- `DOC_TRANS_MODEL`

### 8.2 我改了 `.env`，为什么没生效

优先检查：

- 你改的是不是 `DocTranslationTool.exe` 同目录下的 `.env`
- 有没有把程序放在 zip 压缩包里直接运行
- 有没有只拷贝 `exe`，却没把整个目录一起带上

### 8.3 Anthropic 兼容接口还是连不上

优先检查：

- `DOC_TRANS_API_FORMAT` 是否已经设为 `anthropic`
- `DOC_TRANS_BASE_URL` 是否多写了 `/messages`
- `DOC_TRANS_ANTHROPIC_VERSION` 是否被改成了服务端不接受的值

### 8.4 大文档翻译慢，或者中途报限流

先不要急着把参数调大，反而应该先调小：

- 把 `DOC_TRANS_PARALLEL_BATCHES` 调低
- 必要时把 `DOC_TRANS_BATCH_SIZE` 也调低
- 保留同一个输出目录重试

## 9. 如果只想演示流程，不想占用真实额度

可以临时切到离线演示模式：

```env
DOC_TRANS_PROVIDER=mock
DOC_TRANS_MODEL=mock-demo
```

这个模式下：

- 不会调用真实 API
- 可以演示 GUI、分段、回填和输出流程
- 输出内容只是模拟结果，不适合正式使用

演示完记得切回真实配置。
